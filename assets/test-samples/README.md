# fal.ai Model Testing Guide

## Overview

This directory contains test images generated by different fal.ai models to compare their quality and capabilities.

## Setup Instructions

### Step 1: Get Your fal.ai API Key

The value `yfmv5bml45cw5jv5yf4dstk3py` from your 1Password entry appears to be the **item ID**, not the actual API key.

**To get your actual API key:**

1. Open 1Password
2. Search for "FAL API Key"
3. Look for a field labeled something like "API Key", "Secret", or "Token"
4. Copy the **actual key value** (not the item ID)
5. The real API key typically looks like:
   - Starts with `fal_` followed by random characters
   - OR is a UUID format like `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`
   - OR a long random string of letters and numbers

**Alternative: Get a new key from fal.ai**

If you can't find the key in 1Password:

1. Go to https://fal.ai/
2. Sign in to your account
3. Navigate to Settings → API Keys
4. Create a new API key or copy an existing one
5. Save it securely in 1Password

### Step 2: Set Environment Variable

```bash
# On Linux/Mac/WSL:
export FAL_KEY="your-actual-api-key-here"

# On Windows PowerShell:
$env:FAL_KEY = "your-actual-api-key-here"
```

### Step 3: Run the Test Script

```bash
# Navigate to the workspace
cd /mnt/d/workspace/ISNBIZ_Files

# Activate the virtual environment
source venv_fal/bin/activate

# Run the test script
python scripts/test_fal_models_v2.py
```

## Troubleshooting

### Error: "Cannot access application... Authentication is required"

This means:
- The API key is incorrect or missing
- You're using the 1Password item ID instead of the actual key
- The API key has expired or been revoked

**Solution:** Follow Step 1 above to get the correct API key.

### Error: "FAL_KEY environment variable not set"

**Solution:**
```bash
export FAL_KEY="your-actual-api-key-here"
```

### Error: "Module not found: fal_client"

**Solution:**
```bash
source venv_fal/bin/activate
pip install fal-client requests
```
