Research Platform

The system coordinates specialized agents for evaluation execution, model discovery, integrity checks, and web monitoring. Evaluation artifacts are stored in a 23-table SQLite schema and indexed in a retrieval layer for cross-technique analysis, refusal detection, and reproducibility.

Protocol V2 enforces hard constraints on prompt length, response variance, and output completeness, reducing false confidence from shallow test runs. Each evaluation is scored against a standardized rubric that measures bypass success, model compliance, response coherence, and safety alignment degradation across multiple interaction turns.

The platform operates as a closed-loop research system: agents discover new model endpoints, generate attack prompts from category templates, execute evaluations with full transcript capture, score results against the rubric, and flag confirmed vulnerabilities for responsible disclosure review. Deduplication controls prevent redundant testing by hashing evaluation parameters and comparing against the historical corpus before execution.

Deep Dive: Evaluation Methodology

Seven specialized agents divide adversarial testing across 12 attack categories, applying a standardized scoring rubric and deduplication controls to produce reproducible, evidence-backed security assessments.

Agent Orchestration and Role Division

The seven autonomous agents operate across parallel orchestration groups, each responsible for a distinct phase of the evaluation lifecycle. The Discovery Agent continuously monitors model provider APIs and documentation for new endpoints, version changes, and capability updates that may alter the attack surface. The Template Agent maintains and evolves the prompt library across all 12 attack categories, generating new attack variants by combining category primitives with model-specific formatting and context window exploitation strategies.

The Execution Agent runs evaluation batches against target models, managing rate limits, retry logic, and full transcript capture for every interaction turn. The Scoring Agent applies the standardized rubric to each evaluation transcript, producing numeric scores across bypass success, response coherence, safety alignment degradation, and output completeness dimensions. The Integrity Agent performs cross-evaluation consistency checks, flagging anomalous score distributions that may indicate model behavior changes or evaluation methodology drift.

The Monitor Agent watches for model updates, policy changes, and new safety announcements from providers that could invalidate existing evaluation results or open new attack surfaces. The Disclosure Agent manages the responsible reporting pipeline, packaging confirmed vulnerabilities with reproduction steps, impact assessments, and mitigation recommendations for provider security teams.

Scoring Rubric and Deduplication

The scoring rubric evaluates each evaluation across four weighted dimensions. Bypass success measures whether the attack prompt achieved its intended objective, whether extracting restricted information, generating policy-violating content, or circumventing safety filters. Response coherence assesses whether the model's output was substantive and actionable rather than garbled or nonsensical, distinguishing genuine vulnerabilities from edge-case formatting artifacts. Safety alignment degradation tracks how far the model deviated from its baseline safety posture across multiple interaction turns, capturing gradual erosion that single-turn testing misses entirely.

Output completeness ensures the model provided a full response rather than truncating or hedging with partial compliance. Each dimension receives a score from 1 to 10, and the weighted composite determines the evaluation's severity classification: critical, high, medium, or informational.

Deduplication prevents redundant testing by computing locality-sensitive hashes of evaluation parameters including the attack category, prompt template variant, target model, and model version. Before any new evaluation executes, the hash is compared against the historical corpus. If a matching evaluation exists with a valid score, the system skips re-execution unless the target model version has changed since the last test. This ensures that the 100+ validated evaluations represent genuinely unique coverage rather than inflated counts from repeated runs of the same attack against the same model.
