Technical Direction

The stack is implemented around Playwright-core plus rebrowser controls, with modules for proxy bridge, fingerprint shaping, humanization behavior, and external service checks. The architecture was expanded with DNS leak controls, Decodo zip targeting, and verification tooling designed to reduce detection risk while preserving runtime observability.

Account-kit directories provide project-specific onboarding and operational procedures, making execution reusable across distinct properties. Each kit encapsulates the full lifecycle from account creation through warmup to steady-state maintenance, with target-specific selectors, timing profiles, and verification checkpoints baked into portable configuration files.

The proxy bridge layer abstracts provider complexity behind a unified interface that handles geotargeting, rotation policies, and health scoring. Operators select a geographic target and quality threshold; the bridge handles provider negotiation, sticky session management, and automatic failover when a proxy endpoint degrades below acceptable latency or detection thresholds.

Deep Dive: Browser Automation Architecture

Phantom Browser layers evasion engineering on top of Playwright-core through rebrowser controls, a purpose-built proxy bridge, and a multi-stage verification pipeline that validates stealth posture before any operational action executes.

Playwright-core and Rebrowser Integration

The automation layer is built on Playwright-core rather than full Playwright to avoid bundled browser binaries and retain precise control over the Chromium build used at runtime. Rebrowser patches are applied at launch to neutralize common webdriver detection vectors including navigator.webdriver flags, CDP runtime signatures, and iframe sandboxing artifacts that bot detection services fingerprint.

Each browser context is launched with a synthesized fingerprint profile that covers user agent string, viewport dimensions, WebGL renderer strings, canvas noise injection, AudioContext properties, and timezone-locale consistency. The fingerprint generator draws from a validated distribution of real browser configurations so that each session presents statistically plausible hardware and software characteristics rather than generic defaults that detection systems flag as synthetic.

Context isolation ensures that each account operates in a fully independent browser state with dedicated cookies, local storage, IndexedDB, and service worker registrations. Session state is serialized to disk after each operation and restored on subsequent launches, maintaining login persistence and warmup continuity across execution cycles without cross-contamination between accounts.

Proxy Bridge and Verification Pipeline

The proxy bridge abstracts multiple upstream providers behind a unified routing layer. Operators specify geographic targets at the country, state, or city level, and the bridge resolves the request against available provider inventories, selecting endpoints that meet latency and quality thresholds. Decodo zip-level targeting enables precise geographic alignment for location-sensitive account operations where IP geolocation must match the account's registered address.

Before any operational action fires, the verification pipeline runs a multi-point check sequence. DNS leak tests confirm that all DNS resolution routes through the proxy rather than leaking through the host system. IP quality scoring queries external reputation databases to reject endpoints flagged for abuse, datacenter origin, or VPN detection. WebRTC leak checks verify that the browser context does not expose the real IP through STUN requests. Only after all checks pass does the session proceed to execute the account-kit workflow.

The humanization layer injects realistic behavioral noise into every interaction. Mouse movements follow Bezier curve interpolation with variable velocity profiles. Scroll actions use momentum-based physics with randomized deceleration. Keystroke timing varies per character with realistic inter-key delays that model actual typing cadence. These behavioral overlays make automated sessions statistically indistinguishable from human-driven browser usage in timing analysis.

Differentiation and Moat

Layered Evasion Engineering

Five independent evasion layers -- proxy, fingerprint, DNS, behavior, and verification -- operate in concert so that no single detection vector can compromise an entire session. Each layer is independently testable and upgradeable without disrupting the others.

Pre-Flight Verification

The verification pipeline rejects compromised sessions before any operational action executes, preventing wasted cycles and account exposure. DNS leak tests, IP reputation checks, and WebRTC audits run automatically on every session launch.

Portable Account Kits

Account-kit directories encapsulate target-specific workflows, selectors, timing profiles, and verification checkpoints into portable units that can be developed, tested, and deployed independently across different target platforms without modifying core infrastructure.
